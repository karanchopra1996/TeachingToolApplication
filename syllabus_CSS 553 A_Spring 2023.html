Learning Outcomes

  • Be able to name and describe software architecture concepts, methods, and best practices
  • Be able to demonstrate knowledge of a software architecture research
  • Be able to select and justify the use of software architecture styles or patterns for a given situation
  • Be able to apply software architecture styles or patterns on a small but realistic project
  • Be able to use CASE tools in creating, visualizing, and analyzing software architecture artifacts
